# 재고조사 작업자의 생산성 및 오류율 분석을 통한 문제점 및 원인파악

## 프로젝트 개요
물류센터 재고 검수 과정에서 발생하는 작업 오류 데이터를 분석하여  
작업자별 오류율, 재고 불일치 패턴, 작업 효율성을 분석한 프로젝트입니다.

본 프로젝트에서는 현장의 SBC와 CC 결과 데이터를 유사하게 직접 생성하여, 실제 현장에서 도출 할 수 있는 작업자별 KPI를 설계하고 
통합 분석하여 작업자별 생산성과 오류 책임의 로직을 구현했습니다.

## 주요 분석 내용

### 1. 작업자별 오류율 분석
본 프로젝트는 센터 내 재고조사 데이터를 기반으로:

- 작업자별 생산성
- 작업자별 오류율
- 작업 품질
- 작업 효율성
- 과부하 여부
- 교육 필요 대상
을 분석하기 위해 SQL 기반 KPI 분석쿼리를 구축하였습니다.

## SQL 분석 

| 지표          |       의미               |
| ------------- | ------------------------- |

|생산성 관련 지표 (얼마나 빠른가?)|
| 총 작업량      | 작업자가 처리한 총 물량 (높을수록 현장기여도 높음)    |
| 총 소요시간    | 작업에 소요된 총 시간       |
| UPH           | 시간당 처리량 (생산성 지표/작업자의 순수속도,숙련도 판가름) |
| HTP           | 상품 1개를 처리하는데 걸리는 시간당 처리량 (생산성 지표/ UPH와 반비례, 공정설계시 '개당단가','표준시간'을 잡는 기초 데이터) |
| 전체 작업 건수 | 작업 단위 기준 건수         |

|품질 및 정확도 지표 (얼마나 정확한가?)|
| 총 오류 건수   | 오류 발생 횟수 (작업중 발생한 오류의 총합)    |
| 오류율_퍼센트  | 총 오류건수 / 전체작업건수        |
| 오류율_순위    | 오류가 많은 순서        |

|기여도 및 리스크 지표 (누가 핵심인가?)|
| 누적_작업량  | 총 오류건수 / 전체작업건수        |
| 누적_비중_퍼센| 작업량이 많은 순서대로 비중을 합친 값입니다. (파레토 분석의 핵심)     
                /의미: 80% 선 안에 들어오는 인원들이 사업장 전체 물량의 대부분을 쳐내고 있는 '핵심 엔진'입니다. 이들이 이탈하면 공정의 80%가 멈춘다는 리스크를 시각화   |
| 전체_평균_작업량  |전체 평균(956.80)보다 얼마나 더 혹은 덜 일했는지를 보여줍니다. 성과급이나 인센티브 산정의 기준    |

|종합 평가 및 유형 (어떻게 관리할 것인가?)|
| 작업량_평가  | 평균을 기준으로 '평균 이상'과 '평균 이하'를 자동 분 |
| 작업자_유형  | UPH와 오류율을 결합한 최종 판단 |
